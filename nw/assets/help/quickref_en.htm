Quick Reference Guide

This is the quick reference guide for novelWriter. It briefly lists the key features. The full documentation is available online at novelwriter.readthedocs.io.

How it Works

novelWriter is designed to be a simple and to-the-point application for writing novels. It is not a fancy word processor, and doesn't want to be one either.

All of the pieces of your project is kept in the project treeview in the main window. The files are organised into novel files and notes, which can be cross-referenced and displayed in the Outline tab in the main window.

The text editor panel allows for editing one file at a time, formatted with a simplified markdown syntax and with some additional features specially designed for novels. Spell checking is available through third party libraries.

At any time, the current document being edited, or any other document in the project, can be viewed in the additional view panel. It is not open by default, but can be activated by selecting a file in the treeview, and selecting Document › View Document from the menu, or simply pressing CtrlR.

Markdown Syntax

Only a subset of markdown syntax is supported in the editor, with a few special additions that are convenient for structuring and cross-referencing novels. All supported features are listed in this section.

Headers

Four levels of headers are supported:

Code Description Shortcut
# Title Text Heading Level 1 (Partition, Title) Ctrl1
## Title Text Heading Level 2 (Chapter) Ctrl2
### Title Text Heading Level 3 (Scene) Ctrl3
#### Title Text Heading Level 4 (Section) Ctrl4

The text in parentheses indicate the structure level these headings are treated as in novel files. The level of the heading indicates what the following text is to be considered as. Whether it is the start of a partition, chapter, scene, or a section within a scene. For note files, the headings have no special meaning. In notes, a heading is just a heading.

Header formatting can be removed with the keyboard shortcut Ctrl0.

Text Emphasis

Three types of text emphasis or marking is supported:

Code Description Shortcut
_text_ Emphasised text (italicised) CtrlI
**text** Strongly emphasised text (bold) CtrlB
~~text~~ Strikethrough text Ctrl-

Some additional rules:

  1. The emphasis and strikethrough formatting tags do not allow spaces between the words and the tag itself. That is, **text** is valid, **text ** is not.
  2. More generally, the delimiters must be on the outer edge of words. That is, some **text in bold** here is valid, some** text in bold** here is not.
  3. If using both ** and _ to wrap the same text, the underscore must be the inner wrapper. This is due to the underscore also being a valid word character, so if they are on the outside, they violate rule 2.

Comments

Comments are supported, which is not standard in regular markdown. There are two type of comments available:

  • Regular comments are lines starting with the character %. Such a line is ignored by the word counter, and can be filtered out when the project is exported.
  • Synopsis comments are a special type of comments used to populate the Outline View. They are comments where the first word is Synopsis: (remember the :). They can also be filtered out when the project is exported.

You can turn a regular line into a comment with the keyboard shortcut Ctrl/, and remove it with Ctrl0.

Example:

% This is a regular comment.
% Synopsis: This is a synopsis style comment.

Tags and References

Tags and references are used to cross-reference novel and note files, or note and other note files. Here is a quick summary.

Tags and references use a keyword/value format indicated by the @ character as the first character on the line. The general format is @keyword: value1, value2. The : after the keyword is mandatory, and multiple values are separated with a , character.

Possible keywords are as follows:

Keyword Description
@tag Sets the reference tag for a text section
@pov References a character as the point-of-view character
@char References further non-POV characters
@plot References one or more plot or subplot
@time References one or more time event
@location References one or more plot location
@object References one or more plot object
@entity References one or more entity in the plot
@custom References one or more custom plot element

Project Layout

Projects are structured into a set of top level special folders called root folders. The content is then added to these root folders as files, optionally organised into subfolders. All structure elements are created via the Project menu.

The organisation of files within the root folders are for the user's convenience only. When the project files are assembled into a single documents, only the order of the files is considered. Subfolders are ignored. Files can be reordered by dragging and dropping them.

This section briefly describes the top level structure, and the types of files that the user can choose from.

Root Folders

The most important of the root folders is the folder for the novel files themselves. Additional root folders are meant for organising notes for outlining the novel project. Root folders can be added from the Project menu. Only empty folders can be deleted.

The different type of root folders are:

Name Description Reference Code
Novel The main folder for the novel's text   N
Plot For notes on the novel's plot and subplots @plot P
Characters For notes on the novel's characters @pov, @char C
Locations For notes on the novel's scene locations @location L
Timeline For notes if the novel has multiple timelines @time T
Objects For notes on special scene objects @object O
Entities For notes on various plot entities @entity E
Custom For any other type of notes @custom C

The “Reference” column indicates which keyword can be used to reference a note in that specific folder.

The “Code” column indicates the short code for the file class of each file in the root folder. These short codes are listed in the Flags column of the project treeview in the main window.

File Layout Types

There is one file layout type for notes, and eight file layout types for novel files. The layout types for novel files are mainly a way for the user to quickly see what the files are intended for as the layout type is shown in the project treeview. But they also have some effect on how the text within the file is formatted on export. See the full documentation for further details.

The file layout types are described briefly in the table below.

Type Description Code
Note This is the one file type for note files Nt
Title Page Intended for the cover page Tt
Plain Page Intended for front matter pages Pg
Book Generic file type for novel text Bk
Partition Intended for pages between partitions Pt
Chapter Intended for main story chapters Ch
Unnumbered Intended for special chapters like a prologue Un
Scene Intended for main story scenes Sc

The “Code” column indicates the short code for the file type of each file. These short codes are listed in the Flags column of the project treeview in the main window.

Note that the structure of the novel text itself, like what is a chapter and what is a scene, is not dictated by the file type, but by the header level in the files. This is done so the user can freely choose how they want to split up the project into files, and freely change it in the process by splitting or merging files.

Special Characters and Symbols

A set of special characters and symbols can be inserted from the Insert menu in the main window. These symbols can also be inserted with a double combination of keyboard shortcuts. To activate insert mode, press CtrlK, followed by the key combination for the desired symbol. The options are listed in the menu itself.

Auto-Replace Characters

A few options for auto-replace as you type is available. These can be controlled from the Preferences. These include replacing a series of two or three regular hyphens with short and long dash, respectively. In addition, triple dots can be replaced by the ellipsis character.

The editor also supports automatic conversion from straight quotes to a user-defined set of quote symbols. The default setting is English 99 and 66 style quotes.

Note that the algorithm that detects whether a left or right style quote symbol is needed is fairly basic and may not always make the correct guess. In such cases, use the insert menu options to insert the correct quote character instead.
