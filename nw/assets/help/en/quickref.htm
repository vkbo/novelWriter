Markdown Syntax

Headers

Four levels of headers are supported:

Code Description Shortcut
# Title Text Heading Level 1 (Title) Ctrl1
## Title Text Heading Level 2 (Chapter) Ctrl2
### Title Text Heading Level 3 (Scene) Ctrl3
#### Title Text Heading Level 4 (Section) Ctrl4

The text in parentheses indicate the structure level these headings indicate in novel files. For note files, the headings have no special meaning. See the Project Structure page for more details.

Header formatting can be removed with the keyboard shortcut Ctrl0.

Text Emphasis

Three types of text emphasis or marking are supported

Code Description Shortcut
_text_ Emphasised text (italicised) CtrlI
**text** Strongly emphasised text (bold) CtrlB
~~text~~ Strikethrough text Ctrl-

Some additional rules:

  1. The emphasis and strikethrough formatting tags do not allow spaces between the words and the tag itself. That is, **text** is valid, **text ** is not.
  2. More generally, the delimiters must be on the outer edge of words. That is, some **text in bold** here is valid, some** text in bold** here is not.
  3. If using both ** and _ to wrap the same text, the underscore must be the inner wrapper. This is due to the underscore also being a valid word character, so if they are on the outside, they violate rule 2.

Comments

Comments are supported, which is not standard in regular markdown. There are two type of comments:

  • Regular comments are lines starting with the character %. Such a line is ignored by the word counter, and can be filtered out when the project is exported.
  • Synopsis comments are a special type of comments used to populate the Outline View. They are comments where the first word is Synopsis: (remember the :). They can also be filtered out when the project is exported.

You can turn a regular line into a comment with the keyboard shortcut Ctrl/, and remove it with Ctrl0.

Example:

% This is a regular comment.
% Synopsis: This is a synopsis style comment.
